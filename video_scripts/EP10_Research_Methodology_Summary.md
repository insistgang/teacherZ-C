#### 【01:30-02:30】核心问题的一贯性

**画面**:
- 贯穿各研究的核心问题
- 不同的解法，相同的问题

**旁白**:
> 虽然领域不同，但有一个核心问题贯穿始终：
> 
> 如何从不完整、有噪声的观测中恢复有价值的信息？
> 
> 去噪：从噪声图像恢复干净图像
> 
> 分割：从原始像素恢复语义区域
> 
- 重建：从稀疏采样恢复完整信号
> 
> 方法在变，问题本质不变。这就是研究的「主线」。

**图表需求**:
```
核心问题的统一视角：

┌────────────────────────────────────────────┐
│           不完整/噪声观测                   │
│                 │                          │
│                 ▼                          │
│         ┌───────────────┐                  │
│         │  先验知识     │                  │
│         │  - 稀疏性     │                  │
│         │  - 平滑性     │                  │
│         │  - 几何结构   │                  │
│         └───────┬───────┘                  │
│                 │                          │
│                 ▼                          │
│           优化/学习方法                     │
│                 │                          │
│                 ▼                          │
│           恢复有价值信息                    │
└────────────────────────────────────────────┘

各论文的先验：
- ROF：TV稀疏性
- 紧框架：小波稀疏
- Varifolds：几何测度
- Mamba：数据驱动先验
```

### 【02:30-07:00】方法论提炼

#### 【02:30-03:30】洞察1：先验知识是关键

**画面**:
- 先验的作用示意图
- 无先验 vs 有先验对比

**旁白**:
> 第一条方法论洞察：先验知识是解决病态问题的关键。
> 
> 不论是变分方法还是深度学习，本质上都在利用先验：
> 
> 变分方法：显式定义先验（TV、小波稀疏）
> 
> 深度学习：隐式学习先验（数据驱动）
> 
> 好的研究，往往是找到了更好的先验表达。

**图表需求**:
```
先验知识的作用：

无先验（病态问题）：
观测 → 无穷多解 → ?

有先验（正则化）：
观测 + 先验 → 唯一解

先验类型：
┌────────────────────────────────────────┐
│ 显式先验（变分方法）                    │
│ - TV稀疏性                             │
│ - 小波稀疏                             │
│ - 几何测度                             │
├────────────────────────────────────────┤
│ 隐式先验（深度学习）                    │
│ - 卷积的局部性                         │
│ - 自注意力的全局性                     │
│ - 数据分布                             │
└────────────────────────────────────────┘

研究启示：
找到更好的先验 = 找到更好的解
```

#### 【03:30-04:30】洞察2：跨学科借鉴

**画面**:
- 跨学科借鉴的案例
- 从A领域到B领域的方法迁移

**旁白**:
> 第二条洞察：最好的创新往往来自跨学科借鉴。
> 
> ROF模型来自偏微分方程；
> 
> 紧框架来自调和分析；
> 
> Varifold来自微分几何；
> 
> CUR分解来自数值线性代数。
> 
> 深入一个领域，但不要局限在一个领域。

**图表需求**:
```
跨学科借鉴案例：

┌───────────────────┬────────────────────┐
│ 源领域            │ 应用               │
├───────────────────┼────────────────────┤

│ 调和分析          │ 紧框架血管分割     │

│ 微分几何          │ Neural Varifolds   │

│ 控制理论          │ Mamba状态空间模型  │
└───────────────────┴────────────────────┘

方法：
1. 深入理解一个数学工具
2. 找到它的本质特性
3. 迁移到新问题
4. 适配和改进
```

#### 【04:30-05:30】洞察3：问题驱动 vs 方法驱动

**画面**:
- 两种研究范式的对比
- 最佳平衡点

**旁白**:
> 第三条洞察：问题驱动和方法驱动要平衡。
> 
> 纯粹问题驱动——容易陷入增量式改进，缺乏理论深度；
> 
> 纯粹方法驱动——容易变成「拿着锤子找钉子」，脱离实际需求。
> 
> 最佳状态是：深刻理解问题，掌握丰富的工具箱，灵活选择和创造方法。

**图表需求**:
```
研究范式对比：

问题驱动               方法驱动
   │                      │
   │    最佳平衡区        │
   │   ╔══════════╗       │
   │   ║  ★       ║       │
   │   ║          ║       │
   │   ╚══════════╝       │
   │                      │
   ▼                      ▼
增量改进              脱离实际

最佳实践：
1. 从真实问题出发
2. 深入理解问题本质
3. 掌握多种方法工具
4. 选择/创造合适方法
5. 验证实际效果
```

#### 【05:30-06:30】洞察4：从简单到复杂

**画面**:
- 简单模型到复杂模型的演进
- ROF → TV正则化家族

**旁白**:
> 第四条洞察：从简单模型开始，逐步复杂化。
> 
> 几乎所有重要工作都遵循这个模式：
> 
> 先在一个简化设定下证明核心思想有效；
> 
> 然后逐步处理更复杂的情况；
> 
> 最后扩展到实际应用。
> 
> 不要一开始就追求完美，先让核心想法「跑起来」。

**图表需求**:
```
从简单到复杂的演进：

ROF模型（1992）
    │
    ├─→ 矢量TV（彩色图像）
    │
    ├─→ 非局部TV（纹理）
    │
    ├─→ TV-L1（椒盐噪声）
    │
    └─→ 深度TV（学习先验）

研究策略：
1. 选择一个简化设定
2. 在这个设定下验证核心想法
3. 证明/分析理论性质
4. 逐步扩展到复杂情况
5. 最终处理真实数据

避免：
一开始就追求完美 → 永远完成不了
```

### 【07:00-11:00】跨领域洞察

#### 【07:00-08:00】数学与深度学习的融合

**画面**:
- 传统方法 vs 深度学习
- 融合的趋势

**旁白**:
> 从变分方法到深度学习，是不是意味着数学没用了？
> 
> 恰恰相反！深度学习的很多进展都在重新发现数学。
> 
> ResNet = 常微分方程求解器
> 
- 扩散模型 = 随机微分方程
> 
> NeRF = 隐式神经表示（函数逼近）
> 
> 理解数学，才能理解深度学习的本质。

**图表需求**:
```
数学与深度学习的联系：

│ ResNet           │ ODE求解器              │
│ NeRF             │ 函数逼近               │
│ 扩散模型          │ 随机微分方程           │

│ Mamba            │ 状态空间模型           │

趋势：
```

### 【11:00-14:00】研究生涯启示

#### 【11:00-12:00】选择研究问题

**画面**:
- 好问题的特征
- 如何找到好问题

**旁白**:
> 如何选择研究问题？
> 
> 好问题有几个特征：
> 
> 1. 重要——解决它会有实际影响
> 
> 2. 可行——现有工具能有所进展
> 
> 3. 有趣——你自己愿意投入时间
> 
> 4. 有空间——不是已经被完全解决
> 
> 最好的问题往往在「重要但看起来很难」和「看起来简单但有深意」之间。

**图表需求**:
```
研究问题选择矩阵：

              容易
               │
    ┌──────────┼──────────┐
    │          │          │

    │  跳过    │  可以做  │
困难├──────────┼──────────┤容易
    │          │          │
    │  最重要   │  琐碎   │
    │  优先    │  跳过   │
    │          │          │
    └──────────┼──────────┘
               │
             困难

最佳选择：
重要 + 有一定挑战 + 你有独特洞察
```
