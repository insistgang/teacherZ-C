## 三、嵌套采样算法详解

### 3.1 算法原理

**核心思想**: 通过逐步收缩先验体积来估计证据

**算法流程**:
```
嵌套采样 (Nested Sampling):

初始化:
├── 从先验 p(x) 中采样 N 个活跃点
└── 设置证据估计 Z = 0

迭代过程:
for i = 1, 2, ...:
    1. 找出当前活跃点中似然最小的点 L_min
    2. 估计先验体积 X_i ≈ exp(-i/N)
    3. 更新证据: Z += L_min * ΔX
    4. 用新点替换最小似然点(保持约束 L > L_min)

终止条件:
└── 达到最大迭代次数或证据收敛
```

### 3.2 近端嵌套采样 [4-08]

**核心方法**: Proximal Nested Sampling

**创新点**:
- 引入近端算子处理约束问题
- 适用于带正则化的贝叶斯推断
- 高维问题的高效采样

**数学形式**:
```
近端算子:
prox_{λf}(x) = argmin_u {f(u) + (1/2λ)||u-x||²}

应用于嵌套采样:
- 处理非光滑先验
- 加速收敛
```

### 3.3 数据驱动先验嵌套采样 [4-09]

**核心方法**: Data-Driven Priors

**方法论**:
```
数据驱动先验构建:

历史数据/训练数据
    ↓
学习先验分布 p(x|θ)
    ↓
嵌套采样结合数据驱动先验
    ↓
更精确的后验推断
```

**优势**:
- 利用已有数据信息
- 改善先验设定
- 提高采样效率

### 3.4 嵌套采样算法对比

| 算法变体 | 核心特点 | 适用场景 |
|----------|----------|----------|
| 标准嵌套采样 | 基础证据估计 | 通用贝叶斯推断 |
| 近端嵌套采样 | 处理约束/正则化 | 带约束优化问题 |
| 数据驱动嵌套采样 | 学习先验 | 有历史数据场景 |
