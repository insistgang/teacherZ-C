### 旁白文本

> 「彩色图像分割听起来简单，实则困难重重。
>
> **第一个挑战**：RGB颜色空间的三个通道高度相关——当光线变化，三个通道同时变化。这意味着，仅靠RGB很难区分真正的颜色差异。
>
> **第二个挑战**：实际图像往往不是完美的——噪声、模糊、信息丢失，这些退化让分割雪上加霜。
>
> **第三个挑战**：分割数量K不确定——用户可能想分3类，也可能想分10类，不想每次都重新计算。
>
> SLaT框架，就是为了解决这三大挑战而生。」

### 旁白文本

> 「现在让我们深入SLaT的三个阶段。

> **Stage 1：平滑/恢复**
>
> 对每个颜色通道独立求解一个凸优化问题。能量泛函包含三项：数据项处理噪声，H¹正则项强制平滑，TV正则项保持边缘。使用Split Bregman算法高效求解。
>
> **Stage 2：维度提升**
>
> 这是SLaT的核心创新！将平滑后的RGB图像转换到Lab颜色空间，然后把RGB和Lab拼接起来，形成一个6维向量。为什么这么做？因为RGB和Lab提供了互补的信息。
>
> **Stage 3：阈值分割**
>
> 在6维特征空间中使用K-means聚类。关键设计：Stage 3与前两阶段完全解耦！想改变K值？只需重跑Stage 3，不用重新计算前两阶段。」

### 算法流程动画

```
SLaT完整流程:

   输入：退化RGB图像 f
          │
          ▼
┌──────────────────────────────────────┐
│  Stage 1: Smoothing (平滑/恢复)       │
│  • 对 R/G/B 通道分别求解凸 M-S 模型   │
│  • 输出：平滑图像 ḡ_RGB ∈ [0,1]³     │
└──────────────────────────────────────┘
          │
          ▼
┌──────────────────────────────────────┐
│  Stage 2: Lifting (维度提升)         │
│  • RGB → Lab 颜色空间转换            │
│  • 拼接：g* = [ḡ_RGB, ḡ_Lab]        │
│  • 输出：6维向量图像                  │
└──────────────────────────────────────┘
          │
          ▼
┌──────────────────────────────────────┐
│  Stage 3: Thresholding (阈值分割)    │
│  • K-means 聚类                      │
│  • (可调整K值，无需重算前阶段)       │
│  • 输出：K个分割区域                 │
└──────────────────────────────────────┘
          │
          ▼
   输出：分割标签图
```

>
